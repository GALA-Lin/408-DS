# p20 09
## 题目
给定三个序列A、B、C，长度均为n，且均为无重复元素的递增序列，请设计一个时间上尽可能高效的算法，逐行输出同时存在于这三个序列中的所有元素。例如，数组A为{1,2,3}，数组B 为{2,3,4}，数组C为{-1,0,2}，则输出 2。

## 思路
### 1. 枚举法
#### 核心逻辑
枚举法是最直观的暴力解法，通过三层嵌套循环遍历三个序列的所有元素组合，逐一判断元素是否同时存在于三个序列中：
1. 遍历序列 A 的每个元素 `A[i]`；
2. 对于每个 `A[i]`，遍历序列 B 的每个元素 `B[j]`；
3. 对于每个 `B[j]`，遍历序列 C 的每个元素 `C[k]`；
4. 若 `A[i] == B[j] == C[k]`，则输出该元素。

#### 复杂度分析
- 时间复杂度：$O(n^3)$（三层嵌套循环，每层最多执行 $n$ 次）；
- 空间复杂度：$O(1)$（仅使用少量临时变量）。

#### 优缺点
- 优点：逻辑最简单，无需利用序列“递增、无重复”的特性，新手易理解；
- 缺点：时间复杂度极高，当 $n$ 较大时（如 $n>1000$），运行效率极低，完全不适合实际场景。

#### 示例代码（核心片段）
```cpp
// 枚举法核心逻辑
void enum_method(int A[], int B[], int C[], int n) {
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            for (int k = 0; k < n; k++) {
                if (A[i] == B[j] && B[j] == C[k]) {
                    cout << A[i] << endl;
                }
            }
        }
    }
}
```

### 2. 哈希表法
#### 核心逻辑
利用哈希表的 $O(1)$ 平均查询特性，将“三层遍历”简化为“三层线性遍历”，核心步骤：
1. 先将序列 A 的所有元素存入哈希表（如 C++ 的 `unordered_set`），建立元素的快速查询映射；
2. 遍历序列 B，筛选出同时存在于 A 哈希表中的元素，存入临时哈希表；
3. 遍历序列 C，筛选出同时存在于临时哈希表中的元素，逐行输出。

#### 复杂度分析
- 时间复杂度：$O(n)$（平均情况）/ $O(n^2)$（最坏情况，哈希冲突极端）；
  解释：哈希表插入/查询的平均时间为 $O(1)$，三层线性遍历总次数为 $3n$，因此平均复杂度为 $O(n)$；若哈希冲突严重（如所有元素哈希值相同），查询退化为 $O(n)$，总复杂度变为 $O(n^2)$。
- 空间复杂度：$O(n)$（需要存储两个哈希表，最多占用 $2n$ 空间）。

#### 优缺点
- 优点：时间复杂度远优于枚举法，无需利用序列“递增”特性，适配无序序列场景；
- 缺点：需要额外的空间存储哈希表，且存在哈希冲突导致效率退化的风险，实际运行效率低于三指针法（哈希计算、冲突处理有隐性开销）。

#### 示例代码（核心片段）
```cpp
#include <unordered_set>
// 哈希表法核心逻辑
void hash_method(int A[], int B[], int C[], int n) {
    unordered_set<int> setA(A, A + n); // 存储A的所有元素
    unordered_set<int> setAB;         // 存储A和B的交集
    for (int j = 0; j < n; j++) {
        if (setA.count(B[j])) {
            setAB.insert(B[j]);
        }
    }
    // 遍历C，输出A、B、C的交集
    for (int k = 0; k < n; k++) {
        if (setAB.count(C[k])) {
            cout << C[k] << endl;
        }
    }
}
```

### 3. 三指针法
#### 核心逻辑
充分利用序列“**递增、无重复**”的特性，通过三个指针同步遍历，避免无效循环，是本题的最优解：
1. 初始化三个指针 `i, j, k` 均指向序列起始位置（索引 0）；
2. 循环遍历，直到任意一个指针超出序列长度：
   - 若 `A[i] == B[j] == C[k]`：输出该元素，三个指针均后移（无重复无需去重）；
   - 若不相等：找到三个元素中的最小值，仅将最小值对应的指针后移（递增序列中，更小的元素不可能再与另外两个序列的后续元素匹配）。

#### 复杂度分析
- 时间复杂度：$O(n)$（严格线性，无最坏情况退化）；
  解释：每个指针最多移动 $n$ 次，总循环次数不超过 $3n$，仅包含数值比较和指针移动（无隐性开销）。
- 空间复杂度：$O(1)$（仅使用 3 个指针变量和少量临时变量，无额外存储）。

#### 优缺点
- 优点：时间复杂度最优（严格 $O(n)$）、空间复杂度最优（$O(1)$），实际运行效率最高；
- 缺点：依赖序列“递增、无重复”的特性，若序列无序则无法使用。

#### 示例代码（核心片段）
```cpp
void three_pointer_method(int A[], int B[], int C[], int n) {
    int i = 0, j = 0, k = 0;
    while (i < n && j < n && k < n) {
        if (A[i] == B[j] && B[j] == C[k]) {
            cout << A[i] << endl;
            i++; j++; k++;
        } else {
            int minNum = min(A[i], min(B[j], C[k]));
            if (A[i] == minNum) i++;
            else if (B[j] == minNum) j++;
            else k++;
        }
    }
}
```

### 补充：三种方法对比表
| 方法       | 时间复杂度       | 空间复杂度 | 核心优势                  | 适用场景                     |
|------------|------------------|------------|---------------------------|------------------------------|
| 枚举法     | $O(n^3)$         | $O(1)$     | 逻辑最简单                | 仅适合 $n$ 极小的场景（如 $n<10$） |
| 哈希表法   | $O(n)$（平均）| $O(n)$     | 不依赖序列有序性          | 序列无序、对空间开销不敏感的场景 |
| 三指针法   | $O(n)$（严格）| $O(1)$     | 时间+空间双最优、效率最高 | 序列递增且无重复的场景（本题最优解） |

### 总结
1. 枚举法仅适用于理解基础逻辑，实际开发中完全不推荐；
2. 哈希表法适配性强（支持无序序列），但有空间开销和效率退化风险；
3. 三指针法充分利用题目中“递增、无重复”的特性，是本题时间和空间最优的解法，优先推荐使用。
