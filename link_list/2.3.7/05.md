# 找出两个链表的公共结点

## 题目
给定两个单链表，试分析找出两个链表的公共结点的思想。  
**说明**：公共结点指两个链表中地址相同的结点（即从该结点开始后续结点完全重合，形成“Y”型结构）。假设两个链表均无环，且长度可能不同。

## 思路
### 1. 暴力法
#### 核心逻辑
最直观的解法：遍历第一个链表的每个结点，对于每个结点，遍历第二个链表，检查是否有相同地址的结点。  
- 外层循环遍历链表A的每个结点`p`；  
- 内层循环遍历链表B的每个结点`q`；  
- 若`p == q`，则找到公共结点，返回该结点；  
- 若遍历结束仍未找到，则返回`NULL`。

#### 复杂度分析
- 时间复杂度：$O(m \times n)$，其中`m`、`n`分别为两个链表的长度，最坏情况下需比较所有结点对。  
- 空间复杂度：$O(1)$，仅使用常数个指针变量。

#### 优缺点
- 优点：逻辑简单，无需额外空间，不依赖链表特性。  
- 缺点：时间复杂度极高，当链表较长时效率低下。

#### 示例代码（核心片段）
```cpp
ListNode* getIntersectionNode_brute(ListNode *headA, ListNode *headB) {
    ListNode *p = headA;
    while (p) {
        ListNode *q = headB;
        while (q) {
            if (p == q) return p;
            q = q->next;
        }
        p = p->next;
    }
    return NULL;
}
```

### 2. 双指针法（交替遍历法）
#### 核心逻辑
利用路径长度相等的原理，无需显式计算长度差。  
- 初始化两个指针`pA`和`pB`分别指向链表A和链表B的头结点；  
- 同步向后移动，当某一指针到达链表末尾时，将其重定向到另一个链表的头结点；  
- 当两个指针相遇时，相遇点即为公共结点（若不存在公共结点，则最终会同时指向`NULL`）。

**原理**：指针`pA`走过的路程为 `lenA + lenB`，指针`pB`走过的路程为 `lenB + lenA`，两者相等。若存在公共结点，则在走过相同步数后，两个指针会同时到达公共结点；若无公共结点，则最终同时为`NULL`。

#### 复杂度分析
- 时间复杂度：$O(m + n)$，每个指针最多遍历两个链表各一次。  
- 空间复杂度：$O(1)$，仅使用两个指针变量。

#### 优缺点
- 优点：时间与空间复杂度均为最优，代码简洁，无需计算链表长度，无额外存储。  
- 缺点：需要理解“路径长度相等”的原理，但实现非常简单。

#### 示例代码（核心片段）
```cpp
ListNode* getIntersectionNode_alternate(ListNode *headA, ListNode *headB) {
    if (!headA || !headB) return NULL;
    ListNode *pA = headA, *pB = headB;
    while (pA != pB) {
        pA = pA ? pA->next : headB;
        pB = pB ? pB->next : headA;
    }
    return pA;
}
```

### 3. 栈法
#### 核心逻辑
由于公共结点之后的结点完全相同，若从链表尾部向前比较，第一个不同的结点的下一个结点即为公共结点。  
- 将两个链表的所有结点分别压入两个栈；  
- 同时弹出栈顶元素，比较是否相同，直到遇到第一个不同的结点，则上一个弹出的结点即为公共结点；  
- 若所有结点都相同，则公共结点为链表头。

#### 复杂度分析
- 时间复杂度：$O(m + n)$，需要两次遍历压栈，一次弹出比较。  
- 空间复杂度：$O(m + n)$，需要两个栈存储所有结点地址。

#### 优缺点
- 优点：思路直观，利用了“尾部对齐”特性。  
- 缺点：需要额外空间，且需引入栈结构。

#### 示例代码（核心片段）
```cpp
#include <stack>
ListNode* getIntersectionNode_stack(ListNode *headA, ListNode *headB) {
    stack<ListNode*> sA, sB;
    ListNode *p = headA;
    while (p) { sA.push(p); p = p->next; }
    p = headB;
    while (p) { sB.push(p); p = p->next; }
    ListNode *prev = NULL;
    while (!sA.empty() && !sB.empty()) {
        if (sA.top() == sB.top()) {
            prev = sA.top();
            sA.pop(); sB.pop();
        } else {
            break;
        }
    }
    return prev;
}
```

### 4. 环检测法
#### 核心逻辑
将链表A的尾部连接到链表B的头部（或反之），构造一个环，然后使用快慢指针检测环的入口，即为公共结点。  
- 先遍历链表A找到尾结点，将其`next`指向链表B的头部，形成环；  
- 用快慢指针（Floyd判环法）找到环的入口；  
- 恢复链表A的尾结点`next`为`NULL`（若需要保持原链表结构）。

#### 复杂度分析
- 时间复杂度：$O(m + n)$，快慢指针遍历一次。  
- 空间复杂度：$O(1)$，仅使用几个指针变量。

#### 优缺点
- 优点：不依赖链表长度，利用了环检测的经典思想。  
- 缺点：需要修改链表结构（临时），且必须恢复，实现较复杂。

#### 示例代码（核心片段）
```cpp
ListNode* getIntersectionNode_cycle(ListNode *headA, ListNode *headB) {
    if (!headA || !headB) return NULL;
    // 找到链表A的尾结点
    ListNode *tailA = headA;
    while (tailA->next) tailA = tailA->next;
    tailA->next = headB;  // 形成环
    // 快慢指针找环入口
    ListNode *slow = headA, *fast = headA;
    bool hasCycle = false;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            hasCycle = true;
            break;
        }
    }
    ListNode *intersection = NULL;
    if (hasCycle) {
        slow = headA;
        while (slow != fast) {
            slow = slow->next;
            fast = fast->next;
        }
        intersection = slow;
    }
    // 恢复原链表
    tailA->next = NULL;
    return intersection;
}
```

### 补充：方法对比表
| 方法       | 时间复杂度       | 空间复杂度 | 核心优势                  | 适用场景                     |
|------------|------------------|------------|---------------------------|------------------------------|
| 暴力法     | $O(mn)$          | $O(1)$     | 逻辑最简单，无额外存储    | 仅适合链表长度极小的场景     |
| 双指针法   | $O(m+n)$         | $O(1)$     | 时间+空间双最优，代码简洁 | 无环链表的首选解法           |
| 栈法       | $O(m+n)$         | $O(m+n)$   | 思路直观，利用后进先出    | 链表长度较小或需保留原顺序时 |
| 环检测法   | $O(m+n)$         | $O(1)$     | 巧妙利用环检测，无需长度差 | 可临时修改链表结构的场景     |

### 总结
1. **暴力法**仅适用于理解基础逻辑，实际开发中不推荐。  
2. **栈法**和**环检测法**提供了不同的思路，可加深对链表结构的理解，但在实际应用中不如双指针法简洁高效。  
3. **双指针法（交替遍历法）** 是时间和空间最优的解法，代码最简洁，无需额外存储，是面试和工程中的首选。  

**最佳实践**：对于无环单链表，优先使用双指针交替遍历法，时间复杂度$O(m+n)$，空间复杂度$O(1)$，代码极为精简。
